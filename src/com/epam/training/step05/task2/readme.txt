Задача 2.  
Создать класс Payment с внутренним классом, с помощью объектов которого можно сформировать покупку из 
нескольких товаров. 

Описание приложения:
Существует класс "Payment" с внутренним классом "Product". Класс "Payment" содержит поля "String id" - 
id чека, "long priceTotal" - общая сумма чека, "Product[] products" - массив экземляров внутреннего
класса "Product", т.е. продуктов, входящих в данный чек. Для добавления/удаления продуктов в чек - 
существует класс "PaymentLogic"; для вывода чека на консоль - существует класс "Viewer"; 
класс "Runner" - демонстрирует работу приложения.
